# Metadata Extraction Framework

Extract structured, type-safe data from RDF graphs using JSON Schema.

## Features

- JSON Schema → SPARQL CONSTRUCT → TypeScript objects
- Prisma-style filters (select/include/omit)
- Pagination with ORDER BY
- Automatic injection prevention
- Circular reference handling

## The Complete Pipeline

The framework orchestrates a 4-stage pipeline to transform JSON Schema into extracted data:

### Stage 1: Schema Normalization

Resolves `$ref` references, applies filters, identifies relationships, attaches pagination metadata.

**Package**: `@graviola/edb-graph-traversal`

### Stage 2: SPARQL Generation

Converts normalized schema to SPARQL CONSTRUCT patterns with SUBSELECTs for pagination.

**Package**: `@graviola/sparql-schema`

### Stage 3: Query Execution

Execute SPARQL against your triple store (your SPARQL client handles this).

### Stage 4: Graph Extraction

Walk the RDF dataset with clownface, extract type-safe objects, apply in-memory pagination if needed.

**Package**: `@graviola/edb-graph-traversal`

## Key Features

### Prisma-Style Filters

Inspired by Prisma ORM, the framework supports intuitive data selection:

```typescript
{
  select: { name: true, email: true },           // Pick specific fields
  include: { friends: { take: 10 } },            // Include relationships
  omit: ["password", "internalId"]               // Exclude sensitive fields
}
```

### Consistent Pagination

ORDER BY support ensures repeatable results, even for blank nodes:

```typescript
{
  friends: {
    take: 10,
    skip: 0,
    orderBy: { name: 'asc' }  // Consistent ordering
  }
}
```

### Injection Prevention

All SPARQL queries are built using `@tpluscode/sparql-builder`, automatically escaping all user input and preventing injection attacks.

### Type Safety

The entire pipeline is fully typed with TypeScript, from schema definition to extracted objects.

## Core Packages

### 1. @graviola/edb-graph-traversal

Handles schema normalization and graph extraction.

**Main Exports**:
- `normalizeSchema(schema, filterOptions)` - Resolves refs and applies filters
- `extractFromGraph(iri, dataset, schema, options)` - Extracts objects from RDF

[Go to graph-traversal documentation →](/docs/library-docs-graph-traversal-00-introduction--docs)

### 2. @graviola/sparql-schema

Generates SPARQL CONSTRUCT queries from normalized schemas with type-safe Prisma-style filters.

**Main Exports**:
- `normalizedSchema2construct(iri, schema, options)` - Generates SPARQL patterns
- `buildTypedSPARQLQuery<T>(iri, schema, options)` - Type-safe query builder with filters
- `filterTypedDocument<T>(iri, typeIRI, schema, fetch, options)` - Load entities with filters
- `filterTypedDocuments<T>(typeIRI, schema, selectFetch, constructFetch, options)` - Find entities with filters

[Go to sparql-schema documentation →](/docs/library-docs-sparql-schema-00-introduction--docs)

### 3. @graviola/edb-core-types

Shared TypeScript types used across all packages.

**Key Types**:
- `PaginationOptions` - User-facing pagination API
- `PaginationMetadata` - Internal pagination tracking
- `GraphTraversalFilterOptions` - Filter configuration
- `Prefixes` - SPARQL prefix mappings

## Use Cases
