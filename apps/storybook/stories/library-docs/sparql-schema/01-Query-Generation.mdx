# Query Generation

The `normalizedSchema2construct` function converts a normalized JSON Schema into SPARQL CONSTRUCT and WHERE patterns.

## How It Works

<Mermaid chart={`graph LR
    A[Normalized Schema] --> B[Walk Properties]
    B --> C{Property Type?}

    C -->|Object| E[Nested Pattern]
    C -->|Array| F[Array/SUBSELECT]
    D --> G[CONSTRUCT + WHERE]
    E --> G

## Pattern Types

### Literal Properties

**Schema**:
```typescript
{
  properties: {
    name: { type: "string" }
  }
}
```

**Generated**:
```sparql
CONSTRUCT {
  <subject> :name ?name .
}
WHERE {
  OPTIONAL { <subject> :name ?name }
}
```

### Object Properties

**Schema**:
```typescript
{
  properties: {
    employer: {
      type: "object",
      properties: {
        name: { type: "string" }
      }
    }
  }
}
```

**Generated**:
```sparql
CONSTRUCT {
  <subject> :employer ?employer .
  ?employer :name ?name .
}
WHERE {
  OPTIONAL { <subject> :employer ?employer }
  OPTIONAL { ?employer :name ?name }
}
```

### Array Properties

**Schema** (without pagination):
```typescript
{
  properties: {
    hobbies: {
      type: "array",
      items: { type: "string" }
    }
  }
}
```

**Generated**:
```sparql
CONSTRUCT {
  <subject> :hobbies ?hobbies_0 .
}
WHERE {
  OPTIONAL { <subject> :hobbies ?hobbies_0 }
}
```

## Injection Prevention

All queries use `@tpluscode/sparql-builder` which automatically escapes values:

**User Input**:
```typescript
const maliciousInput = '"; DROP GRAPH <http://example.com>;';
```

**Safe Output** (properly escaped):
```sparql
<subject> :name "\"; DROP GRAPH <http://example.com>;" .
```

The input is treated as a literal string, not executable code.

## Property Name Resolution

The generator uses three rules:

**Rule 1: No colon** → Default prefix
- Input: `"name"`
- Output: `:name`

**Rule 2: Colon + known prefix** → Keep as-is
- Input: `"foaf:name"` with `prefixMap: { foaf: "..." }`
- Output: `foaf:name`

**Rule 3: Colon + unknown prefix** → Full IRI
- Input: `"urn:x121"`
- Output: `<urn:x121>`

### Example

```typescript
normalizedSchema2construct(iri, schema, {
  prefixMap: {
    "": "http://example.com/",
    "foaf": "http://xmlns.com/foaf/0.1/"
  }
});
```

**Property Mappings**:
- `"name"` → `:name` → `<http://example.com/name>`
- `"foaf:name"` → `foaf:name` → `<http://xmlns.com/foaf/0.1/name>`
- `"urn:isbn:123"` → `<urn:isbn:123>`

## Variable Naming

Variables are generated from property names:

- `name` → `?name`
- `firstName` → `?firstName`
- `friends[0]` → `?friends_0`
- `friends[0].name` → `?name_1`

The SPARQL query builder ensures variables are unique.

## OPTIONAL vs Required

All properties generate `OPTIONAL` patterns by default:

```sparql
OPTIONAL { <subject> :name ?name }
```

**Why OPTIONAL?**
- Properties may not exist in the graph
- Partial data is still useful
- Matches JSON Schema optional properties

## Helper Functions

The generator uses helper functions for readability:

```typescript
function createOptionalWherePattern(subject, predicate, object) {
  return sparql`OPTIONAL { ${subject} ${predicate} ${object} }`;
}

function createRequiredWherePattern(subject, predicate, object) {
  return sparql`${subject} ${predicate} ${object} .`;
}
```

## Complete Example

**Input**:
```typescript
const schema = normalizeSchema({
  type: "object",
  properties: {
    name: { type: "string" },
    age: { type: "number" },
    employer: {
      type: "object",
      properties: {
        name: { type: "string" }
      }
    }
  }
});

const result = normalizedSchema2construct(
  "http://example.com/person/1",
  schema,
  {
    prefixMap: { "": "http://schema.org/" }
  }
);
```

**Generated SPARQL**:
```sparql
PREFIX : <http://schema.org/>

CONSTRUCT {
  <http://example.com/person/1> :name ?name .
  <http://example.com/person/1> :age ?age .
  <http://example.com/person/1> :employer ?employer .
  ?employer :name ?name_1 .
}
WHERE {
  OPTIONAL { <http://example.com/person/1> :name ?name }
  OPTIONAL { <http://example.com/person/1> :age ?age }
  OPTIONAL { <http://example.com/person/1> :employer ?employer }
  OPTIONAL { ?employer :name ?name_1 }
}
```

## Return Value

```typescript
{
  constructPatterns: SparqlTemplateResult[],  // For CONSTRUCT clause
  wherePatterns: SparqlTemplateResult[],      // For WHERE clause
  paginationMetadata: Map<string, {           // Pagination tracking
    take?: number,
    skip?: number,
    orderBy?: OrderByClause,
    source: "query"
  }>
}
```

## Building the Complete Query

```typescript
import { sparql } from "@tpluscode/sparql-builder";

const { constructPatterns, wherePatterns } = normalizedSchema2construct(...);

// Add PREFIX declarations
const prefixes = Object.entries(prefixMap)
  .map(([prefix, iri]) => `PREFIX ${prefix}: <${iri}>`)
  .join("\n");

// Build complete query
const query = sparql`
  ${prefixes}
  
  CONSTRUCT {
    ${constructPatterns}
  }
  WHERE {
    ${wherePatterns}
  }
`;

console.log(query.toString());
```

## Performance Tips

- Cache generated patterns when schema doesn't change
- Use filters to reduce query size
- Apply pagination at query stage for large arrays
