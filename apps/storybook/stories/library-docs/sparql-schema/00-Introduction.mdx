# @graviola/sparql-schema

The `@graviola/sparql-schema` package generates SPARQL CONSTRUCT queries from normalized JSON Schemas. It's the bridge between your data model and the RDF triple store.

## Installation

```bash
npm install @graviola/sparql-schema
# or
yarn add @graviola/sparql-schema
# or
bun add @graviola/sparql-schema
```


## Quick Start

### Basic SPARQL Generation

```typescript
import { normalizedSchema2construct } from "@graviola/sparql-schema";
import { normalizeSchema } from "@graviola/edb-graph-traversal";

// Step 1: Normalize schema
const schema = {
  type: "object",
  properties: {
    name: { type: "string" },
    age: { type: "number" }
  }
};

const normalized = normalizeSchema(schema);

// Step 2: Generate SPARQL patterns
const { constructPatterns, wherePatterns } = normalizedSchema2construct(
  "http://example.com/person/1",
  normalized,
  {
    prefixMap: {
      "": "http://example.com/",
      "foaf": "http://xmlns.com/foaf/0.1/"
    }
  }
);

// Step 3: Build complete query
import { sparql } from "@tpluscode/sparql-builder";

const query = sparql`
  PREFIX : <http://example.com/>
  PREFIX foaf: <http://xmlns.com/foaf/0.1/>
  
  CONSTRUCT {
    ${constructPatterns}
  }
  WHERE {
    ${wherePatterns}
  }
`;

console.log(query.toString());
```

**Generated SPARQL**:
```sparql
PREFIX : <http://example.com/>
PREFIX foaf: <http://xmlns.com/foaf/0.1/>

CONSTRUCT {
  <http://example.com/person/1> :name ?name .
  <http://example.com/person/1> :age ?age .
}
WHERE {
  OPTIONAL { <http://example.com/person/1> :name ?name }
  OPTIONAL { <http://example.com/person/1> :age ?age }
}
```

### Full Pipeline Helper

For convenience, use the `constructAndExtract` helper:

```typescript
import { constructAndExtract } from "@graviola/sparql-schema";

const result = await constructAndExtract(
  "http://example.com/person/1",
  schema,
  sparqlClient,
  {
    filterOptions: {
      include: {
        friends: { take: 10, orderBy: { name: 'asc' } }
      }
    },
    prefixMap: {
      "": "http://example.com/"
    }
  }
);

// Result: Extracted JavaScript object
console.log(result);
```

## Main Exports

### normalizedSchema2construct

Generates SPARQL CONSTRUCT and WHERE patterns from a normalized schema.

```typescript
function normalizedSchema2construct(
  iri: string,
  normalizedSchema: NormalizedSchema,
  options?: {
    prefixMap?: Prefixes;
  }
): ConstructResult
```

**Returns**:
```typescript
{
  constructPatterns: SparqlTemplateResult[],
  wherePatterns: SparqlTemplateResult[],
  paginationMetadata: Map<string, PaginationMetadata>
}
```

[Learn more about Query Generation →](/docs/library-docs-sparql-schema-query-generation--docs)

### constructAndExtract

End-to-end helper that normalizes → generates SPARQL → executes → extracts.

```typescript
async function constructAndExtract(
  iri: string,
  schema: JSONSchema7,
  sparqlClient: SparqlClient,
  options?: ConstructAndExtractOptions
): Promise<any>
```

**Features**:
- Automatic schema normalization
- SPARQL generation
- Query execution
- Data extraction
- All in one function!

## Key Features

### Injection-Safe Query Building

All queries are built using `@tpluscode/sparql-builder`, which:

✅ Escapes all user input automatically  
✅ Validates RDF terms  
✅ Prevents SPARQL injection  
✅ Handles special characters safely  

### Pagination with SUBSELECT

For paginated arrays, the generator creates optimized SUBSELECT queries:

```sparql
WHERE {
  OPTIONAL { {
    SELECT ?friends_0 WHERE {
      <http://example.com/person/1> :friends ?friends_0 .
      OPTIONAL { ?friends_0 :name ?name }
    }
    ORDER BY ASC(?name)
    LIMIT 10
  } }
  OPTIONAL { ?friends_0 :name ?name_1 }
}
```

[Learn more about SUBSELECT →](/docs/library-docs-sparql-schema-subselect--docs)

### ORDER BY Support

Ensures consistent pagination, especially for blank nodes:

```typescript
{
  friends: {
    take: 10,
    orderBy: [
      { age: 'desc' },
      { name: 'asc' }
    ]
  }
}
```

**Generated**:
```sparql
ORDER BY DESC(?age) ASC(?name)
```

[Learn more about ORDER BY →](/docs/library-docs-sparql-schema-pagination-orderby--docs)

### Prefix Support

Supports SPARQL prefixes for compact queries:

```typescript
{
  prefixMap: {
    "": "http://example.com/",
    "foaf": "http://xmlns.com/foaf/0.1/",
    "schema": "http://schema.org/"
  }
}
```

**Usage**:
```typescript
// Property "foaf:name" → foaf:name (compact)
// Property "name" → :name (default prefix)
// Property "http://ex.com/prop" → <http://ex.com/prop> (full IRI)
```


## Pattern Generation

The generator creates three types of patterns:

### CONSTRUCT Patterns

Define which triples to extract:

```sparql
CONSTRUCT {
  <subject> :property ?variable .
}
```

### WHERE Patterns

Define how to match triples:

```sparql
WHERE {
  OPTIONAL { <subject> :property ?variable }
}
```

### SUBSELECT Patterns

For paginated arrays:

```sparql
WHERE {
  OPTIONAL { {
    SELECT ?item WHERE {
      <subject> :items ?item .
    }
    ORDER BY ASC(?sortKey)
    LIMIT 10
  } }
}
```

## Property Name Resolution

The generator resolves property names to IRIs:

**No colon** → Default prefix:
- `"name"` → `:name`

**With colon + known prefix** → Leave as-is:
- `"foaf:name"` → `foaf:name` (resolved by SPARQL engine)

**With colon + unknown prefix** → Full IRI:
- `"urn:x121"` → `<urn:x121>`

## Pagination Source Marking

When pagination is applied at the query stage, it's marked with `source: "query"`:

```typescript
{
  paginationMetadata: Map {
    "friends" => {
      take: 10,
      orderBy: { name: 'asc' },
      source: "query"  // ← Marked!
    }
  }
}
```

This prevents double-pagination during extraction.

## Performance Considerations

### Query Complexity

- Each property adds patterns to CONSTRUCT and WHERE
- Nested objects increase query size
- Arrays with pagination add SUBSELECTs

**Tip**: Use filters to limit properties included.

### Triple Store Optimization

Most SPARQL engines optimize:
- `OPTIONAL` patterns
- `LIMIT` and `OFFSET`
- `ORDER BY` with indexed properties

### Caching

SPARQL patterns can be cached if schema doesn't change:

```typescript
const cache = new Map();

function getCached(iri, schema) {
  const key = `${iri}:${JSON.stringify(schema)}`;
  if (!cache.has(key)) {
    cache.set(key, normalizedSchema2construct(iri, schema));
  }
  return cache.get(key);
}
```
