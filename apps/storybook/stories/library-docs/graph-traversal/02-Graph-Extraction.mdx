# Graph Extraction

Graph extraction is the final stage in the metadata extraction pipeline. It walks an RDF dataset using a normalized schema to produce a type-safe JavaScript object.

## The extractFromGraph Function

```typescript
import { extractFromGraph } from "@graviola/edb-graph-traversal";

const result = extractFromGraph(iri, dataset, normalizedSchema, options);
```

### Signature

```typescript
function extractFromGraph(
  iri: string,
  dataset: DatasetCore,
  normalizedSchema: NormalizedSchema,
  options?: ExtractionOptions
): any
```

### Parameters

**iri** (string)
- The subject IRI to start extraction from
- Example: `"http://example.com/person/1"`

**dataset** (DatasetCore)
- RDF/JS dataset containing triples
- Usually from SPARQL CONSTRUCT result

**normalizedSchema** (NormalizedSchema)
- Normalized schema (from `normalizeSchema`)
- All `$ref` references must be resolved

**options** (optional)
- `baseIRI`: Base IRI for expanding property names
- `prefixMap`: Prefix mappings for compact IRIs
- `logger`: Custom logger for debugging

### Returns

Extracted JavaScript object matching the schema.

## Using Clownface

The extractor uses [clownface](https://github.com/zazuko/clownface), a powerful RDF graph navigation library.

### Creating a Pointer

```typescript
import clownface from "clownface";
import { namedNode } from "@rdfjs/data-model";

const pointer = clownface({ dataset })
  .namedNode("http://example.com/person/1");
```

###

 Navigating Properties

```typescript
// Get single value
const name = pointer.out(namedNode("http://schema.org/name")).value;

// Get multiple values
const friends = pointer.out(namedNode("http://schema.org/knows"));
for (const friend of friends.values) {
  console.log(friend);
}
```

### Following Relationships

```typescript
// Follow to related node
const employer = pointer.out(namedNode("http://schema.org/worksFor"));

// Check if it's a NamedNode
if (employer.term?.termType === "NamedNode") {
  console.log("Employer IRI:", employer.value);
}
```

## Literal Extraction

Extracts and coerces literal values according to the schema type.

### String Extraction

```typescript
// RDF: <person> :name "John"
// Schema: { type: "string" }
// Result: "John"
```

### Number Extraction

```typescript
// RDF: <person> :age "30"^^xsd:integer
// Schema: { type: "number" }
// Result: 30 (number, not string)
```

### Boolean Extraction

```typescript
// RDF: <person> :active "true"^^xsd:boolean
// Schema: { type: "boolean" }
// Result: true (boolean)
```

### Type Coercion

The extractor automatically coerces values:

```typescript
function extractLiteral(value: string, schema: JSONSchema7): any {
  if (schema.type === "number") {
    return parseFloat(value);
  }
  if (schema.type === "integer") {
    return parseInt(value, 10);
  }
  if (schema.type === "boolean") {
    return value === "true" || value === "1";
  }
  return value; // string
}
```

## Object Extraction

Extracts objects by following relationships.

### NamedNode Objects

Named nodes have IRIs and get an `@id` property:

```typescript
// RDF: <person> :employer <http://example.com/company/1>
//      <http://example.com/company/1> :name "Acme Corp"

// Result:
{
  employer: {
    "@id": "http://example.com/company/1",
    name: "Acme Corp"
  }
}
```

### BlankNode Objects

Blank nodes are anonymous and have no `@id`:

```typescript
// RDF: <person> :address _:b1
//      _:b1 :city "New York"

// Result:
{
  address: {
    city: "New York"
    // No @id property
  }
}
```

### Recursion Depth

The extractor respects `maxRecursion` from the schema:

```typescript
{
  maxRecursion: 3,  // Stop after 3 levels
}
```

If depth limit is reached, recursion stops.

## Array Extraction

Extracts array values with optional pagination.

### Simple Arrays

```typescript
// RDF: <person> :hobby "reading"
//      <person> :hobby "coding"

// Schema: { type: "array", items: { type: "string" } }

// Result:
{
  hobby: ["reading", "coding"]
}
```

### Arrays of Objects

```typescript
// RDF: <person> :friend <person/2>
//      <person> :friend <person/3>
//      <person/2> :name "Jane"
//      <person/3> :name "Bob"

// Result:
{
  friends: [
    { "@id": "http://example.com/person/2", name: "Jane" },
    { "@id": "http://example.com/person/3", name: "Bob" }
  ]
}
```

### Pagination

Arrays can be paginated at two stages:

**Query-Stage Pagination** (already applied):
```typescript
// x-pagination: { take: 10, source: "query" }
// → Use all items in dataset (already limited by SPARQL)
```

**Extraction-Stage Pagination** (apply now):
```typescript
// x-pagination: { take: 10, skip: 5, source: "extraction" }
// → Slice array: items.slice(5, 15)
```

**Double-Pagination Prevention**:
```typescript
const pagination = schema["x-pagination"];
if (pagination?.source === "query") {
  // Already paginated - use all
  return items;
} else if (pagination) {
  // Apply pagination
  const { skip = 0, take } = pagination;
  return items.slice(skip, skip + take);
}
return items;
```

## Complex Types

### anyOf

Tries each schema in order until one matches:

```typescript
// Schema: { anyOf: [{ type: "object" }, { type: "null" }] }
// RDF: <person> :data null

// Tries object schema first (fails)
// Tries null schema second (matches)
// Result: null
```

### oneOf

Similar to `anyOf`, but expects exactly one match:

```typescript
// Schema: { oneOf: [{ type: "string" }, { type: "number" }] }
// RDF: <person> :value "42"

// Result: "42" (string schema matches)
```

## Extraction Context

The extractor maintains context during traversal:

```typescript
interface ExtractionContext {
  dataset: DatasetCore;          // RDF dataset
  baseIRI: string;                // Base IRI for properties
  prefixMap: Prefixes;           // Prefix mappings
  depth: number;                  // Current recursion depth
  maxRecursion: number;           // Max depth allowed
  visited: Set<string>;           // Prevent cycles
  logger: Logger;                 // Logging
}
```

### Cycle Prevention

The extractor tracks visited nodes:

```typescript
if (context.visited.has(nodeIRI)) {
  // Already visited - stop recursion
  return { "@id": nodeIRI };
}
context.visited.add(nodeIRI);
```

## Property Name Expansion

Property names are expanded to full IRIs:

```typescript
// Schema property: "name"
// Base IRI: "http://schema.org/"
// Expanded: "http://schema.org/name"
```

### Prefix Support

```typescript
// Schema property: "foaf:name"
// Prefix map: { foaf: "http://xmlns.com/foaf/0.1/" }
// Expanded: "http://xmlns.com/foaf/0.1/name"
```

## Examples

### Example 1: Simple Object

**RDF**:
```turtle
<http://example.com/person/1> :name "John" .
<http://example.com/person/1> :age "30"^^xsd:integer .
```

**Schema**:
```typescript
{
  type: "object",
  properties: {
    name: { type: "string" },
    age: { type: "number" }
  }
}
```

**Result**:
```typescript
{
  "@id": "http://example.com/person/1",
  name: "John",
  age: 30
}
```

### Example 2: Nested Object

**RDF**:
```turtle
<person/1> :name "John" .
<person/1> :employer <company/1> .
<company/1> :name "Acme Corp" .
```

**Result**:
```typescript
{
  "@id": "http://example.com/person/1",
  name: "John",
  employer: {
    "@id": "http://example.com/company/1",
    name: "Acme Corp"
  }
}
```

### Example 3: Array with Pagination

**RDF**:
```turtle
<person/1> :friend <person/2> .
<person/1> :friend <person/3> .
<person/1> :friend <person/4> .
<person/2> :name "Jane" .
<person/3> :name "Bob" .
<person/4> :name "Alice" .
```

**Schema**:
```typescript
{
  properties: {
    friends: {
      type: "array",
      items: { /* ... */ },
      "x-pagination": {
        take: 2,
        skip: 0
      }
    }
  }
}
```

**Result** (only 2 items):
```typescript
{
  friends: [
    { "@id": "http://example.com/person/2", name: "Jane" },
    { "@id": "http://example.com/person/3", name: "Bob" }
  ]
}
```

## Error Handling

The extractor handles common errors gracefully:

**Missing properties**:
```typescript
// If property not in graph, returns undefined
{ name: "John", email: undefined }
```

**Invalid types**:
```typescript
// Logs warning, returns raw value
logger.warn("Expected number, got string");
```

**Deep recursion**:
```typescript
// Stops at maxRecursion, returns stub
{ "@id": "...", /* no nested properties */ }
```

## Logging

The extractor supports custom loggers:

```typescript
const logger = {
  debug: (msg: string, data?: any) => console.log(msg, data),
  warn: (msg: string, data?: any) => console.warn(msg, data),
  error: (msg: string, data?: any) => console.error(msg, data),
};

extractFromGraph(iri, dataset, schema, { logger });
```

## Performance Tips

### Dataset Size

Extraction is O(triples), so smaller datasets are faster:

- Use query-stage pagination for large arrays
- Limit recursion depth
- Apply filters at normalization stage

### Clownface Efficiency

Clownface is optimized for:
- Following relationships (`.out()`, `.in()`)
- Checking node types
- Iterating values

Avoid:
- Converting entire dataset to objects
- Unnecessary dataset clones

## Type Safety

The extraction is fully typed:

```typescript
import type { ExtractionOptions, ExtractionContext } from "@graviola/edb-graph-traversal";
```
