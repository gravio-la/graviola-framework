# @graviola/edb-graph-traversal

Schema normalization (Stage 1) and graph extraction (Stage 4).

```bash
bun add @graviola/edb-graph-traversal
```

## Quick Start

### Schema Normalization

```typescript
import { normalizeSchema } from "@graviola/edb-graph-traversal";
import { JSONSchema7 } from "json-schema";

const schema: JSONSchema7 = {
  type: "object",
  properties: {
    name: { type: "string" },
    friends: {
      type: "array",
      items: { $ref: "#/$defs/Person" }
    }
  },
  $defs: {
    Person: {
      type: "object",
      properties: {
        name: { type: "string" },
        age: { type: "number" }
      }
    }
  }
};

// Normalize with filters
const normalized = normalizeSchema(schema, {
  include: {
    friends: {
      take: 10,
      orderBy: { name: 'asc' }
    }
  }
});

// Result: Flat schema with all $refs resolved
console.log(normalized);
```

### Graph Extraction

```typescript
import { extractFromGraph } from "@graviola/edb-graph-traversal";
import datasetFactory from "@rdfjs/dataset";

// Assume you have an RDF dataset
const dataset = datasetFactory.dataset();
// ... populate dataset from SPARQL CONSTRUCT result

// Extract object from graph
const person = extractFromGraph(
  "http://example.com/person/1",  // Subject IRI
  dataset,                          // RDF dataset
  normalized,                       // Normalized schema
  {
    baseIRI: "http://schema.org/"
  }
);

// Result: Type-safe JavaScript object
console.log(person);
// {
//   "@id": "http://example.com/person/1",
//   name: "John",
//   friends: [{ name: "Jane", age: 30 }, ...]
// }
```

## Main Exports

### normalizeSchema

Converts a JSON Schema with `$ref` references into a flat, resolved schema.

```typescript
function normalizeSchema(
  schema: JSONSchema7,
  filterOptions?: GraphTraversalFilterOptions
): NormalizedSchema
```

**Features**:
- Resolves all `$ref` references recursively
- Applies select/include/omit filters
- Identifies relationships
- Attaches pagination metadata
- Handles circular references safely

[Learn more about Schema Normalization →](/docs/library-docs-graph-traversal-schema-normalization--docs)

### extractFromGraph

Walks an RDF dataset using a normalized schema to extract a TypeScript object.

```typescript
function extractFromGraph(
  iri: string,
  dataset: DatasetCore,
  normalizedSchema: NormalizedSchema,
  options?: {
    baseIRI?: string;
    prefixMap?: Record<string, string>;
    logger?: Logger;
  }
): any
```

**Features**:
- Type-safe extraction using clownface
- Literal value coercion (strings, numbers, booleans, dates)
- Follows relationships (NamedNodes and BlankNodes)
- Applies in-memory pagination when needed
- Respects pagination source markers

[Learn more about Graph Extraction →](/docs/library-docs-graph-traversal-graph-extraction--docs)

## Key Concepts

### Normalized Schema

A `NormalizedSchema` is a JSON Schema with:
- All `$ref` references resolved inline
- No `$ref` pointers remaining
- Additional metadata attached:
  - `_normalized: true` flag
  - `_propertyMetadata` with relationship information
  - `x-pagination` on array properties

**Example**:
```typescript
{
  type: "object",
  properties: {
    name: { type: "string" },
    friends: {
      type: "array",
      items: {
        type: "object",
        properties: {
          name: { type: "string" }
        }
      },
      "x-pagination": {
        take: 10,
        orderBy: { name: 'asc' }
      }
    }
  },
  _normalized: true,
  _propertyMetadata: {
    friends: {
      isRelationship: true,
      isArray: true,
      itemType: "object"
    }
  }
}
```

### Property Metadata

Each property in a normalized schema has associated metadata:

```typescript
{
  isRelationship: boolean;  // Is this a relationship to another entity?
  isArray: boolean;          // Is this an array property?
  itemType: string;          // Type of array items (if array)
}
```

This metadata guides the extraction process.

### Clownface

The extractor uses [clownface](https://github.com/zazuko/clownface), a graph traversal library for RDF/JS, to navigate RDF datasets.

**Example**:
```typescript
import clownface from "clownface";

// Create a pointer to a node
const pointer = clownface({ dataset })
  .namedNode("http://example.com/person/1");

// Follow relationships
const name = pointer.out(":name").value;
const friends = pointer.out(":friends");
```

Clownface provides a fluent API for graph navigation that the extractor uses internally.

## Filter Options

The package supports Prisma-style filters:

### Select

Pick specific fields:

```typescript
{
  select: {
    name: true,
    email: true
  }
}
```

### Include

Include relationships with optional pagination:

```typescript
{
  include: {
    friends: true,           // Include all
    posts: {                 // Include with pagination
      take: 10,
      skip: 0,
      orderBy: { date: 'desc' }
    }
  }
}
```

### Omit

Exclude specific fields:

```typescript
{
  omit: ["password", "internalId"]
}
```

### Global Options

```typescript
{
  includeRelationsByDefault: false,  // Don't auto-include relationships
  defaultPaginationLimit: 50,        // Default LIMIT for arrays
  maxRecursion: 4                    // Max depth for $ref resolution
}
```

[Learn more about filtering →](/docs/architecture-key-concepts--docs#prisma-style-filters)

## Pagination

The package supports pagination at the extraction stage:

```typescript
const normalized = normalizeSchema(schema, {
  include: {
    posts: {
      take: 20,
      skip: 10,
      orderBy: { createdAt: 'desc' }
    }
  }
});

// If source !== "query", pagination is applied during extraction
const result = extractFromGraph(iri, dataset, normalized);
```

**Note**: If pagination was already applied at the SPARQL query stage (marked with `source: "query"`), the extractor will skip in-memory pagination to prevent double-pagination.

[Learn more about pagination →](/docs/architecture-key-concepts--docs#pagination-options)

## Performance

- Cache normalized schemas for reuse
- Set appropriate `maxRecursion` (default: 4)
- Use query-stage pagination for large arrays
