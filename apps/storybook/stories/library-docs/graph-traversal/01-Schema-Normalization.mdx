# Schema Normalization

Schema normalization is the first critical stage in the metadata extraction pipeline. It converts a JSON Schema with `$ref` references into a flat, fully-resolved schema ready for SPARQL generation and graph extraction.

## What is Schema Normalization?

JSON Schema allows defining reusable schema fragments using `$ref` references:

```json
{
  "type": "object",
  "properties": {
    "author": { "$ref": "#/$defs/Person" }
  },
  "$defs": {
    "Person": {
      "type": "object",
      "properties": {
        "name": { "type": "string" }
      }
    }
  }
}
```

**The Problem**: References must be resolved before we can:
- Generate SPARQL queries
- Walk RDF graphs
- Apply filters consistently

**The Solution**: Normalize the schema by resolving all `$ref` pointers inline.

## The normalizeSchema Function

```typescript
import { normalizeSchema } from "@graviola/edb-graph-traversal";

const normalized = normalizeSchema(schema, filterOptions);
```

### Signature

```typescript
function normalizeSchema(
  schema: JSONSchema7,
  filterOptions?: GraphTraversalFilterOptions
): NormalizedSchema
```

###

 Parameters

**schema** (JSONSchema7)
- The input JSON Schema, which may contain `$ref` references
- Must be a valid JSON Schema Draft 7 schema

**filterOptions** (optional)
- Prisma-style filter options to apply during normalization
- Includes: `select`, `include`, `omit`, `includeRelationsByDefault`, etc.

### Returns

A `NormalizedSchema` object with:
- All `$ref` references resolved
- Filters applied
- Property metadata attached
- Pagination configuration

### Phase 1: Resolve All $refs

Recursively resolves all `$ref` pointers up to a configurable depth.

**Input**:
```typescript
{
  type: "object",
  properties: {
    author: { $ref: "#/$defs/Person" },
    friends: {
      type: "array",
      items: { $ref: "#/$defs/Person" }
    }
  },
  $defs: {
    Person: {
      type: "object",
      properties: {
        name: { type: "string" },
        age: { type: "number" }
      }
    }
  }
}
```

**Output** (refs resolved):
```typescript
{
  type: "object",
  properties: {
    author: {
      type: "object",
      properties: {
        name: { type: "string" },
        age: { type: "number" }
      }
    },
    friends: {
      type: "array",
      items: {
        type: "object",
        properties: {
          name: { type: "string" },
          age: { type: "number" }
        }
      }
    }
  }
}
```

### Phase 2: Extract Property Metadata

Analyzes each property to determine:
- Is it a relationship?
- Is it an array?
- What type are the items?

**Example Metadata**:
```typescript
{
  author: {
    isRelationship: true,
    isArray: false,
    itemType: "object"
  },
  friends: {
    isRelationship: true,
    isArray: true,
    itemType: "object"
  }
}
```

**Relationship Detection**:
- Objects with an `@id` property are relationships
- Schema hint: `x-relationship: true`
- Configurable via options

### Phase 3: Apply Filters

Applies Prisma-style filters to determine which properties to include.

**Filter Options**:
```typescript
{
  select: { name: true },      // Pick fields
  omit: ["internalId"],          // Exclude fields
  include: {                      // Include relationships
    friends: {
      take: 10,
      orderBy: { name: 'asc' }
    }
  }
}
```

**Result**: Schema contains only selected/included properties.

## Handling Circular References

Circular references are common in schemas:

```typescript
{
  type: "object",
  properties: {
    name: { type: "string" },
    friends: {
      type: "array",
      items: { $ref: "#" }  // ← Points back to root!
    }
  }
}
```

Without protection, this would expand infinitely!

### Max Recursion Depth

The normalizer uses `maxRecursion` to prevent infinite expansion:

```typescript
normalizeSchema(schema, {
  maxRecursion: 3  // Stop after 3 levels
});
```

**Result**:
```typescript
Person {
  name: string,
  friends: Person[] {
    name: string,
    friends: Person[] {
      name: string,
      friends: Person[] {
        // Stopped at depth 3
      }
    }
  }
}
```

### Depth Counter

The normalizer tracks depth during resolution:

```typescript
function resolveRef(ref, depth) {
  if (depth > maxRecursion) {
    return {}; // Insert stub
  }
  // Resolve reference
  return resolveRef(target, depth + 1);
}
```

## Filter Application

### Select Pattern

**Purpose**: Only include explicitly selected fields.

**Example**:
```typescript
normalizeSchema(schema, {
  select: {
    name: true,
    email: true
  }
});
```

**Result**: Schema only contains `name` and `email` properties.

### Include Pattern

**Purpose**: Control which relationships are included.

**Example**:
```typescript
normalizeSchema(schema, {
  includeRelationsByDefault: false,
  include: {
    friends: {
      take: 10,
      orderBy: { name: 'asc' }
    }
  }
});
```

**Result**:
- Only `friends` relationship is included
- Pagination metadata attached to `friends` property
- Other relationships excluded

### Omit Pattern

**Purpose**: Exclude specific fields.

**Example**:
```typescript
normalizeSchema(schema, {
  omit: ["password", "internalId"]
});
```

**Result**: Schema excludes `password` and `internalId`.

### Filter Precedence

- **omit** is checked first (always excludes)
- **select** is checked next (if specified, only selected fields)
- **include** determines which relationships to fetch

## Pagination Metadata

When pagination is specified in `include` patterns, the normalizer attaches `x-pagination` metadata:

**Input**:
```typescript
{
  include: {
    posts: {
      take: 20,
      skip: 10,
      orderBy: { createdAt: 'desc' }
    }
  }
}
```

**Output** (in normalized schema):
```typescript
{
  properties: {
    posts: {
      type: "array",
      items: { /* ... */ },
      "x-pagination": {
        take: 20,
        skip: 10,
        orderBy: { createdAt: 'desc' }
      }
    }
  }
}
```

This metadata is used by:
- SPARQL generator to create SUBSELECT queries
- Graph extractor to apply in-memory pagination (if needed)

## Normalized Schema Structure

A `NormalizedSchema` has this structure:

```typescript
{
  // Standard JSON Schema fields
  type: "object",
  properties: {
    // All properties (with refs resolved)
  },
  required: ["name"],
  
  // Normalization marker
  _normalized: true,
  
  // Property metadata
  _propertyMetadata: {
    friends: {
      isRelationship: true,
      isArray: true,
      itemType: "object"
    }
  }
}
```

### Special Fields

**_normalized** (boolean)
- Always `true` for normalized schemas
- Indicates normalization is complete

**_propertyMetadata** (Record\<string, PropertyMetadata\>)
- Metadata for each property
- Used to guide extraction

**x-pagination** (on array properties)
- Pagination configuration
- Optional, only present if pagination specified

## Configuration Options

### maxRecursion

Controls depth of `$ref` resolution.

```typescript
{
  maxRecursion: 4  // Default
}
```

**Use Cases**:
- Flat schemas: 1-2
- Moderate nesting: 3-4 (default)
- Deep hierarchies: 5-6

### includeRelationsByDefault

Controls automatic relationship inclusion.

```typescript
{
  includeRelationsByDefault: true  // Default
}
```

**When true**: All relationships included unless explicitly excluded.  
**When false**: Only explicitly included relationships are fetched.

### defaultPaginationLimit

Global default for array pagination.

```typescript
{
  defaultPaginationLimit: 100
}
```

Applied to arrays without explicit `take` value.

## Performance Tips

### Caching Normalized Schemas

Normalized schemas can be cached and reused:

```typescript
const cache = new Map();

function getNormalized(schema, filters) {
  const key = JSON.stringify({ schema, filters });
  if (!cache.has(key)) {
    cache.set(key, normalizeSchema(schema, filters));
  }
  return cache.get(key);
}
```

**Benefits**:
- Avoid re-normalizing same schema
- Faster query generation
- Reduced CPU usage

### Optimizing Max Recursion

Set `maxRecursion` appropriately:

```typescript
// Too low: Missing data
{ maxRecursion: 1 }

// Optimal: Balanced
{ maxRecursion: 3-4 }

// Too high: Slow, large schemas
{ maxRecursion: 10 }
```

## Error Handling

The normalizer throws errors for:

- Invalid JSON Schema
- Missing `$defs` for `$ref`
- Circular references exceeding max depth

**Example**:
```typescript
try {
  const normalized = normalizeSchema(schema);
} catch (error) {
  if (error.message.includes('maximum recursion')) {
    // Handle circular reference
  }
}
```

```typescript
}
```
